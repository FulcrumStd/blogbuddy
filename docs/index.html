No. 0.13 — A Markdown Apparatus

BlogBuddy writes with you.

<bb-expd:add concrete examples> → press ⌘B ⌘B

An idea, sharply put.

BlogBuddy puts the verbs of AI writing — expand, polish, translate, summarise, diagram — directly into your Markdown as tags. The tag is the command.

No tab-switching. No chat window. No copy-paste shuttle. Just syntax that resolves, in place, where you typed it.

I

The premise

a working theory

You are writing Markdown. You want AI help — to expand a thin paragraph, polish a clumsy sentence, translate a section, summon a Mermaid diagram from a paragraph of prose. Most tools yank you out to a chat window, then back, then out again. Context erodes; flow disappears.

BlogBuddy refuses the round trip. It makes AI a piece of syntax. A tag like <bb-expd:focus on practical use> sits in your document like any other Markdown construct. The chord ⌘B ⌘B resolves it. Output streams in, token by token, exactly where the tag was.

That is the entire thesis. Everything that follows — the WYSIWYG editor, the AI-rendered reading view, the configuration ergonomics — is the work of doing this idea well.

One screen, one resolution

how software gets written.

Place the cursor in the paragraph. Press the chord. The tag is removed; the paragraph expands; words arrive as the model writes them. The scope — line, block, document — is inferred from where the tag sits.

II

The vocabulary

eight verbs + a renderer

A small, deliberate set of commands. Each accepts an optional refinement after the colon, e.g. <bb-foo:say more about X>. The shape is the contract — terse syntax, predictable behaviour.

<bb:any instruction> Free-form. Hand BB any task; it works on the current scope. The flexible escape hatch — and the most token-hungry. block / doc
<bb-expd:focus> Expand. Elaborate on existing content while keeping tone and structure. Reads full-document context so additions fit. block
<bb-impv:style> Improve. Polish clarity, grammar, rhythm. Inline rewrites the block; standalone tag rewrites the whole document into *_improved.md. block / doc
<bb-tslt:target language> Translate. Outputs a new file with a language suffix (e.g. myblog_日本語.md) and inserts a link. Code blocks preserved. document
<bb-tldr:style> Summarise. Inline TL;DR for the surrounding text. Tight by default; refinement bends tone. block
<bb-mmd:diagram description> Diagram. Generates a fenced Mermaid block. Auto-picks flowchart · sequence · class · state · ER · gantt · pie. Validated for a recognised prefix. block
<bb-kwd:focus> Keywords. Extracts 8–12 SEO-friendly keywords from the document, scoped to the focus you set. block
<bb-tag> Attribution. Inserts the "created with BB" shield, in the canonical yellow. inline
<bb-render-*> Reader. Four presets and a custom prompt. Generates a full HTML reading artifact in a side panel. Never modifies your source. document
Streaming · defaultOutput appears as the model writes it. Cancel any time.
Context · awareBB reads the surrounding document, not just the selection.

III

The editor

a WYSIWYG that respects Markdown

A built-in rich-text editor for .md files, powered by Milkdown. It reads and writes plain Markdown — no proprietary format, no lock-in. Open from the Explorer with ⌘B.

It is the canvas where the slash menu lives, where AI blocks stream into place, and where frontmatter becomes typed UI rather than YAML you have to remember.

  • Slash menu/ opens block types and every BB command, pre-filled.
  • Inline chord⌘B ⌘B fires any <bb-*> tag without leaving the canvas.
  • Streaming AI blocksResults appear inline as a typed ProseMirror node.
  • Frontmatter as UITyped controls for title · date · tags · author · draft — raw YAML fallback below.
  • Prism highlighting20+ languages, palette tracks VS Code light / dark / high-contrast.
  • IME-safeCJK composition guards auto-save and BB triggers.
  • Arrow ligatures-> → →, => → ⇒; never inside code.
  • Compact saveBullets to -, blank lines collapsed, entities decoded. Git diffs stay quiet.
  • External-change bannerChoose Reload or Keep when the file shifts under your feet.
  • Paste-to-disk assetsFiles write to the document folder (or assetDir) and link as relative paths.
IV

The reader

your markdown, redesigned

Trigger <bb-render-*> and the AI re-interprets your document as a complete HTML "reading artifact" — rendered in a side panel, exported as a self-contained file, never touching your source.

No. I
<bb-render-blog:>

Blog View

Polished article — sticky TOC, callouts, syntax-highlighted code, comfortable measure. The default literary treatment.

No. II
<bb-render-skim:>

Skim Mode

TL;DR at the top, dense visual structure, <details> collapsibles for skippable depth. Built for the impatient.

No. III
<bb-render-expl:>

The Explainer

A teaching artifact — SVG diagrams for concepts, annotated code, "why this matters" sidebars. For tutorials.

No. IV
<bb-render:your prompt>

Custom

Your prompt is the brief. Override the preset entirely and ask for what you actually want. A blank canvas, AI-aided.

Live preview

HTML composes itself in a sandboxed iframe as the AI writes it. Phase indicator, token counter, editable refinement prompt.

Self-contained export

Single-file .html, local images base64-inlined by default. Toggle inlineAssets to keep relative paths.

.bbreader.md style ref

Drop a style guide at the workspace root; every render uses it as authoritative guidance on type, color, components.

V

Quick start

five steps · five minutes

Install

BlogBuddy from the VS Code Marketplace.

Set credentials

blogbuddy.apiKey in settings, or export BLOGBUDDY_API_KEY.

(Optional) base URL

For OpenAI-compatible providers — OpenRouter, DeepSeek, your own proxy.

Pick a model

Run BlogBuddy: Select Model in the palette.

Open · tag · chord

Open any .md, drop a tag, press ⌘B ⌘B.
